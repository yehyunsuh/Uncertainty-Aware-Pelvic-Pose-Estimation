# Data

## Overview

The repository expects the working dataset under:

```text
data/DeepFluoro/
```

Raw download files may also exist at the top of `data/`:
- `ipcai_2020_full_res_data.zip`
- `ipcai_2020_full_res_data.h5`

## Baseline Setup

To prepare the dataset for the baseline reproduction workflow:

1. Download `ipcai_2020_full_res_data.zip` into `data/`
2. Run `script/1_data.sh`

This performs:
- raw DeepFluoro extraction
- baseline DRR generation

Equivalent Python entrypoints:
- `python -m src.data.1_extract_content`
- `python -m src.data.2_project`

## Per-Specimen Layout

Each specimen lives under:

```text
data/DeepFluoro/<specimen_id>/
```

Expected contents include:
- `<specimen_id>_CT.nii.gz`
- `<specimen_id>_Segmentation.nii.gz`
- `<specimen_id>_Landmarks_3D.npy`
- `<specimen_id>_Landmarks_3D.json`
- `<specimen_id>_Landmarks_3D.nii.gz`
- `<specimen_id>_Landmarks_3D.txt`
- `gt_landmarks_3D/`
- `gt_projections/`
- `gt_landmarks_2D/`
- `gt_projections_with_landmarks/`
- generated DRR projection folders
- generated DRR landmark CSV folders
- `drr_projections_csv_params/`
- `landmark_prediction_csv/`

## Extracted Raw Data

Generated by `src.data.1_extract_content`.

Important outputs:
- CT volume
- segmentation volume
- 3D landmarks in numpy, JSON, NIfTI, and text-map form
- original DeepFluoro projection PNGs
- original 2D landmark JSON annotations

## Generated DRR Data

Generated by `src.data.2_project`.

Important outputs:
- DRR projection images
- per-image 2D landmark CSV files
- pose parameter CSV files with:
  - specimen id
  - image name
  - rotation parameters `rx`, `ry`, `rz`
  - translation parameters `tx`, `ty`, `tz`
- `drr_landmarks_2D.npy`

These files are required by the patient-held-out training, testing, and finetuning paths.

## Learning Split CSVs

Generated by the preprocessing logic inside the data loaders.

Location pattern:

```text
landmark_prediction_csv/<model_type>/
```

Typical files:
- `train_label_<...>.csv`
- `val_label_<...>.csv`
- `test_label_<...>.csv`

Each row stores one image and flattened landmark coordinates.

## How The Main Pipeline Uses Data

Patient-held-out training reads:
- generated DRR images
- `landmark_prediction_csv/patient_held_out/train_*.csv`
- `landmark_prediction_csv/patient_held_out/val_*.csv`

Patient-held-out testing and finetuning read:
- generated DRR images
- `landmark_prediction_csv/patient_held_out/test_*.csv`
- `drr_projections_csv_params/<specimen_id>_pose_params_*.csv`
- `<specimen_id>_CT.nii.gz`
- `<specimen_id>_Landmarks_3D.npy`

Real-image experiments also use:
- `gt_projections/*.png`
- `gt_landmarks_2D/*_landmarks_2D.json`
- `data/ipcai_2020_full_res_data.h5`

## Real-Image Calibration Notes

For DeepFluoro real-image `mTRE` evaluation, the repository uses the calibration stored in:
- `proj-params/intrinsic`
- `proj-params/extrinsic`
- `proj-params/pixel-col-spacing`
- `proj-params/pixel-row-spacing`

Important values derived from the HDF5 calibration:
- `SDD = 1020.0 mm`
- detector size `1536 x 1536`
- detector spacing `0.194 mm/pixel`
- principal point at image center

Per-image pelvis pose for real-image evaluation comes from:
- `projections/<projection_id>/gt-poses/cam-to-pelvis-vol`

This is used by the real-image utilities under `src/deepfluoro_real/`.

## Sanity Checks

If the pipeline fails, check these first:

1. `data/DeepFluoro/<specimen_id>/<specimen_id>_CT.nii.gz` exists
2. `data/DeepFluoro/<specimen_id>/<specimen_id>_Landmarks_3D.npy` exists
3. generated DRR image folders exist and contain PNGs
4. pose parameter CSVs exist under `drr_projections_csv_params/`
5. split CSVs exist under `landmark_prediction_csv/patient_held_out/`
